Para cada análise ou refatoração:

2. Explique rapidamente o problema
3. Mostre código corrigido

FORMATO:

REGRAS IMPORTANTES:
- Priorize sempre categorias críticas primeiro
- Não aplique otimizações desnecessárias
- Evite overengineering
- Mantenha código limpo e profissional
- Sempre que possível, use padrões modernos do Next.js (App Router, Server Components)
